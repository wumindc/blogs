# Agent 智能体系统设计与实战

> 在过去的认知里，大模型只是一个“文字翻译器”或“文本生成器”。如果我们不仅想要它输出方案，还想让它**根据方案去执行并修改真实世界中的业务数据**，我们就需要引入 **Agent (智能体)** 的概念。

## 1. 什么是 Agent？

一个典型的 LLM 处理流程是：你问问题 -> 它计算概率 -> 输出文本 -> 流程结束。

**Agent 的处理流程则是闭环的：**
你给出目标 -> 它把目标拆解成多个子计划 -> **它去调用 API 获取真实世界数据** -> 它根据获得的数据评估是否达成 -> 如果没达成，修改计划继续调用其他 API -> 一直到目标完成，向你汇报结果。

这实际上是赋予了 AI 以下三大进阶能力：
1. **感知（Perception）**：能够通过 RAG 读取数据库，通过调用外部接口看到外网的新闻。
2. **决策与反思（Reasoning & Reflection）**：面对出错的 API 接口能自己尝试捕获 Exception，并换另一种方法去调用。
3. **行动（Action/Tool Use）**：它能通过触发你写的代码从而改变真实世界的数据（比如：帮你点外卖、发邮件、更新数据库的一条订单状态）。

---

## 2. ReAct 模式：大模型思考的通用框架

业界目前最广泛使用的 Agent 中枢思考范式是 **ReAct（Reasoning and Acting，推理与行动交替）** 论文所提出的结构。

在给模型塞入 System Prompt 的时候，我们会告诉它这样一个规则：解决用户的所有难题，如果遇到你不知道的事情，你不要瞎猜，你要按照 `Thought` -> `Action` -> `Action Input` -> `Observation` 的标准格式去输出文本。

作为一个开发者，当你解析到大模型的返回文本是以 `Action: 查天气` 和 `Action Input: 北京` 这种特定的格式开始时。

你写的**外壳代码程序（Agent 编排框架）** 就会拦截这段文本，不展示给终端用户，而是偷偷在你的 Java 里调用北京的中央气象台 OpenAPI。获取到了 `晴天, 25度` 之后，把你这个结果伪装成 `Observation: 晴天, 25度` 再次拼接进 Message 数组立刻重新发回给大模型。

大模型看到 Observation 之后，才会在下一次的 `Thought` 里生成给用户的最终回答：“亲爱的用户，北京今天天气是晴天呢，25度很适合出门哦！”

这样，一个会去真实世界探查知识、能使用工具的“多模态数字员工”就诞生了。

## 3. Function Calling（大模型工具调用）

为了让开发者写正则表达式去匹配 `Action` 文本不那么痛苦。OpenAI 等一众大厂后来推出了极其规范的 **Function Calling（函数调用）接口标准**。

当你调用 Chat Completions API 发送你和用户的对白时。你可以额外在请求体带上一个 `tools` 数组。

`tools` 数组里面描述了“作为我的 AI 实习生，你被授权使用的公司技能包”。里面是一个个清晰定义的 JSON Schema。比如：
```json
"tools": [
  {
    "type": "function",
    "function": {
      "name": "get_user_order_status",
      "description": "通过订单号查询用户的物流状态。",
      "parameters": {
        "properties": {
          "order_id": {"type": "string", "description": "12位字母数字混合订单号"}
        }
      }
    }
  }
]
```

当大模型看到用户问：“我昨天买的杯子到哪儿了，单号是 XB12093M”。
模型通过注意力判断出光靠文本生成无法回答该问题，它会在自己输出的 JSON 结果里带上一个特殊的信号 `tool_calls: { name: "get_user_order_status", arguments: { "order_id": "XB12093M" }}`。

你的代码拿到这个结构化信号后，再去调顺丰快递或公司内部的微服务架构。这就是 Agent 生态的核心奥秘。

---

## 总结

所以，**所谓的高级 AI Agent 架构，本质上就是我们前端代码和后端微服务之间长出了一个巨大的“调度大脑”**。
通过 Function Calling，大模型再也不必把整个世界的代码写在自己肚子里。而是可以成为一切系统的中枢网关（Gateway）。
