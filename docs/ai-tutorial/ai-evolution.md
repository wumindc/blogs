# 从人工智能到生成式 AI：一条更适合工程师理解的演进路线

> 如果你是这两年才认真看 AI，很容易产生一种错觉：世界像是突然换了版本，昨天还在调模型参数，今天已经在讨论 Agent 和推理模型了。
> 但把时间线拉长一点就会发现，今天这波生成式 AI 不是凭空冒出来的，它更像前两轮技术积累终于汇到了一起：先是人工设计特征，后来是网络自己学特征，再后来是大规模预训练把很多任务揉进了同一种能力底座里。

::: info 这篇文章重点
- 为什么“生成式 AI”不是与过去完全割裂的新事物
- 特征工程、深度学习和预训练大模型分别解决了什么问题
- 为什么规模化训练会带来能力跃迁
:::

## 1. 特征工程时代：人来告诉模型该看什么

在早期机器学习里，真正决定效果的往往不是模型本身，而是人能否设计出有效特征。

例如：

- 文本分类看词频、n-gram
- 推荐系统看点击率、停留时长
- 图像任务看边缘、纹理、局部统计

这种方法的问题在于：

- 高度依赖专家经验
- 迁移成本高
- 面对复杂任务时，手工特征很快碰到上限

## 2. 深度学习时代：让模型自己学特征

深度学习的重要变化是：不再要求人显式定义所有特征，而是让多层网络从数据中逐步学习表示。

这带来了两个直接收益：

- 对复杂模式的表达能力更强
- 在语音、图像、文本等任务上形成更统一的方法论

但这一阶段的大多数系统仍然偏“单任务”：

- 一个模型做分类
- 一个模型做识别
- 一个模型做翻译

系统能力很强，但任务边界依然清晰。

## 3. 预训练大模型时代：先学通用表示，再适配具体任务

大模型最关键的转变，是训练方式的变化。

它不再主要依赖人工标注任务，而是先在海量通用数据上做预训练，学习语言和世界中的统计规律；之后再通过提示词、微调、工具调用、RAG 等方式适应具体任务。

这意味着同一类模型可以做很多事：

- 问答
- 摘要
- 翻译
- 代码生成
- 信息抽取

这不是因为模型“知道所有任务”，而是因为它先学到了一种更通用的表示和生成能力。

## 4. 为什么规模化会带来能力跃迁

大模型时代最重要的经验之一，是“规模”本身会改变能力边界：

- 更多数据
- 更大参数量
- 更强训练算力

当这三者一起增长时，模型会在一些任务上表现出非线性提升。这也是为什么今天的生成式 AI 看起来不像传统 NLP 工具的简单升级，而更像能力级别的跳变。

## 5. 这对工程实践有什么影响

理解这条演进路线，会直接影响你对 AI 系统的判断：

- 模型不是规则引擎，不能期待绝对确定性
- 模型擅长模式泛化，但不擅长天然保存实时事实
- 模型本体越来越通用，但系统落地仍然依赖检索、工具和工程边界

所以今天做 AI 应用，真正重要的往往不是“重新发明一个模型”，而是如何把通用模型能力和外部系统组合起来。

## 小结

从人工智能到生成式 AI，并不是从“不会智能”突然跳到“会思考”，而是：

- 人工特征逐步让位于自动表示学习
- 单任务模型逐步让位于通用预训练模型
- 模型能力越来越强，但系统工程的重要性反而更高

理解这条路线，有助于我们更冷静地看待今天的大模型能力边界。
