# 写代码的新姿势：构建对 AI 极其友好的“语境上下文”

> 为什么你在 B 站看别人演示 Cursor 时如同操控魔法，但轮到自己写公司代码时，AI 生成的结果却总是驴头不对马嘴、不断调用错公司的内部包、疯狂地捏造不存在的基类？
> 核心还是出在**上下文投喂 (Context Prompting)** 上。

在以前，我们写注释是为了防范三个月后不认识代码的自己。
在 AI 时代，**你的注释，不是写给人看的，是专门塞给 AI 吃的。**

---

## 1. 大模型的“词穷”之殇：信息隔离

假设你在做电商系统。你通过 `@` 呼出 AI，跟它说：`“在这个订单类里帮我增加计算总运费的逻辑”`。
大模型面临的绝望在于：
1. 这个工程里有三百个跟“运费”相关的方法，你指的是调用哪一个上游库？
2. 公司在打折包邮时有一个特殊的 `FreeShippingEnum` 配置项，模型根本不知道它的存在，所以它会自己生造一个原生的枚举类型。

这就是最大的问题：**大模型很聪明，但如果你用极其简陋的问题去提问，它就犹如陷入信息茧房，为了完成任务，它只能自嗨式的捏造幻觉。**

---

## 2. 什么是面向 AI 的“代码注释提示学”？

改变你的编码顺序，尝试采用 **“先写顶层注释，再让 AI 接替打工” (Comment-Driven Development)** 的方式。这相当于你在通过极其详尽的注释充当“需求文档”，从而在模型的 Context 窗口里搭建一个严密的舞台。

**反面教材的注释：**
```java
// 计算运费
public BigDecimal computeFee(Order order) { ... }
```

**面向 AI 的高维度注射器：**
```java
/**
 * TODO [Task For AI]: 
 * 根据用户的订单距离和当前是否有促销活动来计算运费。
 * 
 * 业务约束 (Context Constraints):
 * 1. 【非常关键！】请必须调用公司内部通用的仓储包 `com.nihoo.logistics.DistanceUtils.calculate(lon, lat)` 来获取最终的公里数距离，绝对不要自己写一套 Haversine 经纬度估算公式。
 * 2. 如果公里数 < 3，并且 `order.hasVip == true`，直接返回 `BigDecimal.ZERO`。这对应了春节活动的免费条款。
 * 3. 结果请四舍五入保留 2 位小数，返回 `BigDecimal` 类型。
 * 
 * 请顺着上方的方法签名将完整逻辑补全。
 */
public BigDecimal computeShippingFeeForVIP(Order order) {
```
通过这段详尽的需求式、带强烈防错边界的注释作为开端，AI (无论是 Cursor 还是 Copilot) 接下来 `Tab` 补全出的内容，绝对是你梦寐以求的精准逻辑，而且它绝对不敢再瞎造包名了。

## 3. 合理引流工具 (如 Cursor 里的 @ 标记)

当你使用带极其强大的文件系统级别索引的工具时，单单用注释可能还不够。你应该使用类似 `@Docs`, `@Web`, `@Files` 等功能，**把外围极其庞大的相关知识强制扯进它的短期记忆窗口**。

*   **引入相关文件 (@File)：** 
    你在写 A 文件时，你深知 A 文件强依赖了极其庞大复杂的 B 文件。如果 AI 看不到 B 文件的具体实现，它会根据 A 里的形参瞎猜。
    你只需在对话框输入：`@UserMapper.java 请参考它里面的 SQL 映射查询，帮我把这段 Controller 相关接收的 DTO 数据结构对齐。` 这等同于直接用几万个 Token 给它加了个知识库补丁。

*   **引入框架的最新网页文档 (@Docs / @Web)：**
    如果你在写一段 Vue 3 最新的冷门 Composition API，或者公司刚接了一个连谷歌都没收录的全新云服务。
    输入：`@Web 查一下官方最新发布的 API 文档示例。然后按着它的格式帮我重写我的本地 axios 调用逻辑。`。这使得模型直接通过搜索技术，越过了它预训练资料的陈旧限制期（比如它可能依然认为 Vue 需要写很啰嗦的代码）。

---

## 4. 极致的命名规约

AI 的推理（接龙能力）也是看风水认字的。
* 看到 `List<User> list1;` 这种稀烂的命名，AI 在推理几层以后就会出现**注意力的分散与崩盘**。
* 看到 `List<User> validActiveUsers;` 这种极其携带大量隐藏语义信息的命名，AI 的自注意力机制将会像吸铁石一样精准把后面的激活逻辑代码完美写出。

规范严谨、不缩水、带丰富常识词缀的代码命名，不只是方便你的同事阅读，它实际上构成了模型进行无监督学习的最佳输入端。
