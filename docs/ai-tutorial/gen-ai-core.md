# 生成式 AI 的运作核心：大语言模型在想什么？

> 很多人把大语言模型（Large Language Models, LLM）当作一个拥有全世界知识的“谷歌搜索”或者一个无所不知的“智者”。**但这是一个美丽的误解。**

要用好 AI 工具，我们必须拨开它神似人类智能的外衣，去看看它底层的核心机制。作为开发者，理解这个机制是我们写好 Prompt、预判 AI 错误的基石。

---

## 1. 大语言模型的核心本质：文字接龙

如果你只能用一句话来解释目前所有主流大模型（如 ChatGPT、Claude、通义千问）在做什么，那这句话就是：**预测下一个词（Token）的概率。**

这就是它本质的全部。就像我们在手机键盘上打字，输入“今天我”，输入法的候选词会弹出“想去”、“不舒服”、“真开心”。大语言模型本质上就是一个**超级巨大、极其精准的高级输入法自动补全引擎**。

当我们向 AI 提问：`"中国的首都是"`时：
1. 模型内部并**没有**一个内嵌的维基百科去查找答案。
2. 它通过极其庞大的神经网络（参数达到千亿级别），计算出在看到在这段文本后，下一个出现的词概率最高的是哪一个。
3. 它算出了 `"北"` 的概率是 99.8%，于是输出了 `"北"`。
4. 接着，它把 `"中国的首都是北"` 再算一遍，得出下一个词 `"京"` 的概率是 99.9%，于是输出 `"京"`。

这种机制被称为**自回归（Autoregressive）**。哪怕它在帮你写一段复杂的快排算法，或者写一首七言绝句，它的核心机制也是一个个 Token“接龙”算出来的。

---

## 2. 让接龙变成奇迹：Transformer 与注意力机制

如果是简单的接龙，它怎么能理解长篇大论而且不偏题呢？这归功于 2017 年 Google 发布的一篇里程碑论文 **《Attention Is All You Need》** 中提出的核心架构：**Transformer**。

Transformer 的灵魂在于**自注意力机制（Self-Attention）**。

### 什么是注意力机制？（打个比方）
想象你在阅读上面的句子：“*银行不给我贷款，因为他们觉得我穷。*”
当大模型处理到“**他们**”这个词时，注意力机制会让模型将极高的权重分配给前面的“**银行**”这个词。模型不需要硬编码任何语法，它通过海量数据的训练，自己“学会了”词与词之间的长距离依赖关系。

因为它可以极快地（并行计算）并行比对一整段上下文中成千上万个词的互相依赖关系，它所生成的“下一个词”就显得极其符合逻辑和人类的智能直觉。

---

## 3. Token：模型眼中的基本单位

作为一个程序员，你需要了解经常在 API 计费或者长度限制里听到的 **Token** 到底是什么。

大模型并不认识字或者字母。在它眼里，所有人类文本都会被切碎成一个个称为 Token 的数字 ID。
* **英文**：通常一个英文单词可能被当作一个 Token（例如 `"apple"`），或者被拆成多个 Token（例如 `"hamburger"` 可能被拆成 `"ham"`、`"bur"`、`"ger"`）。
* **中文**：通常一个汉字算 1 到 2 个 Token。

> **一个粗略的经验法则**：对于 GPT 系列模型，1 个 Token 大约等于 0.75 个英文单词，或者 0.5 个中文字符。

我们常说的 **上下文窗口（Context Window）** 指的就是模型一次性接收和处理的 Token 最大数量。比如有的模型支持 128K Token，意味着你一次可以喂给它十几万字的 PDF 电子书，然后让它基于这本书继续进行“文字接龙”。

---

## 4. 为什么 AI 会产生“幻觉” (Hallucination)？

了解了“接龙”本质，你就自然明白为什么大模型会一本正经地胡说八道（产生幻觉）了。

1. **没有现实世界的“对错观”**：模型只是根据数学预测什么文本放在这里最“通顺”。如果它的训练数据里存在大量偏见，或者你在 Prompt 里预设了一个错误的前提，它大概率会顺着你的假前提示去拼接出符合逻辑却不符合现实的内容。
2. **知识并非被确切存储**：模型的知识是被隐式地压缩在那千亿个参数之间的概率分布中，而不是存在一张关系型数据库表中。所以问它复杂的数学或冷门事实，它很容易通过概率拼接出看似合理实则错误的结果。

---

## 总结

所以，请始终牢记：**大语言模型是一个基于统计概率的、有着庞大上下文注意力的“高级补全引擎”。** 
它是语言的大师，但不必然是事实的主宰。
