# 什么是 AI 基础设施与大模型基座？

> 前面我们讲了，单张顶配的 NVIDIA H100 GPU 几百万人民币，但对于动辄几千亿参数的 GPT-4 来说，一张卡连模型的一小部分参数都装不下。
> 因此，如何把成千上万张卡“缝合”在一起，成为了 AI 的基建难题。

## 1. 数据中心与算力集群 (Cluster)

要训练一个 GPT-4 规模的“基座大模型（Foundation Model）”，往往需要一万到几万张 A100/H100 高端显卡几个月不间断地狂奔。

这里面最可怕的不是买卡的钱，而是**通信网络与功耗**。

* **NVLink 与 InfiniBand (IB)**：当模型被切成好几块分别放在成百上千台机器上算的时候，机器之间数据交互的延迟简直是致命的。为了解决这个问题，需要比普通家用网线快成百上千倍的光纤网络架构（如 IB 网络），保证它们之间每秒能以 TB 级的数据量互相同步结果。
* **“吃电怪兽”**：一个万卡集群一天跑下来不仅极其耗电，而且会散发惊人热量。传统的风扇已经压不住了，这就逼出了**液冷服务器**的全面普及。

所以，现在的“算力中心”，拼的早就不是单一一块芯片有多强，而是**谁能把这成千上万台机器像一台机器一样稳如泰山地调度好**。稍微短板一点的一个交换机，都可能让几千万一天的电费打了水漂。

## 2. 什么是 MaaS 平台？(Model-as-a-Service)

作为普通应用开发者的我们，是不可能去买万卡集群的。

于是大厂们推出了 **MaaS (模型即服务)** 理念。简单来讲：“我的机器已经花半年时间训练好了一个极其聪明的万能大脑。现在，我不卖机器，我只卖这个大脑的脑力，你按次找我提问，我按词（Token）收你几分钱。”

* 海外的 **OpenAI Platform**、**Anthropic Console**。
* 国内的 **阿里云百炼** (提供通义千问)、**字节火山引擎** (提供豆包等大模型)、**百度千帆**。

这些平台就是最顶尖的 AI 基础设施服务商。你甚至可以在他们的网页控制台上点几个按钮，花上几百元钱，就能用他们上万台机器跑出的庞大模型，为你的企业单独微调（Fine-Tune）一个特定版本出来。

---

## 3. 开源世界的“GitHub” —— Hugging Face

如果你不想寄人篱下，想要自己本地免费部署模型（部署在自己的服务器或带强大显存的电脑上），那你绝对避不开这个网站：**[Hugging Face](https://huggingface.co/)**。

你可以把 Hugging Face 理解为**大模型时代自带权重的 GitHub**。

* 全球的开源大神（比如 Meta 开源的 Llama 3 系列、阿里的 Qwen 开源系列、法国的 Mistral）在耗费千万美金算力炼出模型后，会大方地把那包含几百个 GB 权重的核心文件包免费上传倒 Hugging Face 供大家下载（`git clone`）。
* 同时，它还提供了一个名为 `transformers` 的 Python 神级极简库，让你三行代码就能把网上的模型加载到自己的机器显存里跑起来。
