# 实战：产品经理如何“一句话”构建高仿真系统

在传统的开发流程中，如果产品经理（PM）想要体验或展示一个包含完整交互逻辑和数据存储的原型系统，往往需要等待研发团队长周期的排期与开发。

但现在，有了特定的 AI 技能（Skill）—— **`pm-prototype-builder`**，即使是没有任何技术背景的产品经理，也可以通过自然语言“一句话”快速生成具备**前后端联调**能力的高仿真系统。

## 什么是 pm-prototype-builder？

`pm-prototype-builder` 是一个专门为快速搭建管理系统而设计的 AI Skill。它内置了约定的技术栈（如前端 React + TypeScript + Vite，后端 Node.js/Python 等，以及一套标准的中后台 UI 组件库）和基本开发要求。

你只需要告诉它你的业务需求，它就能自动生成包含视图、列表、表单、甚至模拟接口功能的完整项目代码。

## 实战演示：构建“用户信息管理”

假设我们需要一个用户管理模块。按照传统的高保真原型制作，你可能需要在 Axure 里面画半天线框图和交互跳转。现在，我们只需要一句简单的指令。

### 第一步：输入需求

向启用了该 Skill 的 AI 助手输入以下指令：

> **"pm-prototype-builder：创建一个前后端集成的项目，编写一个用户信息管理功能"**

![输入指令示例](/images/prompt-example.png)

*(注：请将您实际聊天界面的截图保存为 `prompt-example.png` 放到 `docs/public/images/` 目录下)*

### 第二步：验收成果

执行完毕后，你会得到一个立刻可以运行的 Web 系统。让我们来看看它生成了什么：

#### 1. 完善的列表页 (List View)
系统自动生成了标准的中后台列表页面，不仅包含了用户数据的表格展现，还自带了顶部的多条件筛选（关键字、用户名、所属部门）、分页器，以及操作列（详情、编辑、停用、删除）。

![用户列表页](/images/user-list.png)
*(注：请将列表截图保存为 `user-list.png` 放到 `docs/public/images/` 目录下)*

#### 2. 表单与编辑页 (Edit View)
点击“编辑”按钮，会进入用户资料维护页面。表单不仅排版规范，还自动处理了必填项、下拉选择框（性别、所属部门、角色名称、状态）、只读项逻辑等。

![用户编辑页](/images/user-edit.png)
*(注：请将表单截图保存为 `user-edit.png` 放到 `docs/public/images/` 目录下)*

#### 3. 详情页 (Detail View)
点击“详情”按钮，系统还生成了一个信息展示页面。它采用了更加清晰的描述列表（Descriptions）和卡片布局，将用户的核心摘要和基本信息按模块划分，并且在顶部带有状态警告提示（如“当前用户已停用”）。

![用户详情页](/images/user-detail.png)

## 总结

借助 `pm-prototype-builder`，产品经理只需关注**业务属性**和**流程逻辑**。繁琐的技术选型、组件调用、前后端接口定义和联调，都由预设的 Skill 标准化解决。这极大拉近了“想法”和“可运行软件”之间的距离，是敏捷验证需求的神兵利器！
