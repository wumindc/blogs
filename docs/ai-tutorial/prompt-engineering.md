# Prompt 提示词工程指南

> “AI 的能力上限，往往取决于操控它的人提出问题的水平。”
> —— 这就是为什么 **Prompt Engineering (提示词工程)** 成为开发者必须掌握的核心技能。

上一节我们明白了 LLM 是在进行“文字接龙”。我们要做的，就是通过精准地提供前置的语境和限制，把模型逼到一条特定的概率轨道上，让它非产出我们想要的结果不可。

本节我们将系统梳理写出高质量提示词的框架方法。

---

## 1. 高质量 Prompt 的万能公式：RCTF 框架

跟人类沟通时，如果没有交代背景，人类也会懵圈。对 AI 也是一样。一个成熟的、能够稳定输出高质量结果的 Prompt，通常应该包含以下几个核心模块，我们称之为 **RCTF 框架**：

*   **R (Role - 角色设定)**：告诉 AI 它是谁，这能激活它模型内对应专业领域的神经网络知识。
*   **C (Context - 上下文/背景)**：解释为什么做这件事，有哪些前置条件或已知资料。
*   **T (Task - 具体任务)**：清晰、无歧义地描述你需要它做什么。
*   **F (Format - 格式与约束)**：规定以什么样的格式（JSON、表格、Markdown、去除无关寒暄等）输出。

### 反面例子 (小白发问)
> 帮我写一个处理日期字符串的函数。

*(AI 会不知道你要用什么语言、处理什么格式的日期、需不需要考虑时区等，可能会输出一大堆冗长的啰嗦内容)*

### 正面例子 (RCTF 工程师发问)
> **[Role]**
> 你是一个资深的高级 Java 后端开发工程师，精通 JDK 8 之后的各种新特性和最佳实践。
> 
> **[Context]**
> 我们系统现在正在做国外电商订单模块，数据库存放的都是 UTC 时间的长整型时间戳（Long），但前端需要展示 YYYY-MM-DD HH:mm:ss 格式的北京时间。
> 
> **[Task]**
> 请用 Java 编写一个无状态的静态工具方法，接收一个 Long 类型的 UTC 时间戳，把它转换为指定的北京时区的字符串格式。请注意处理输入为空或异常的情况。
> 
> **[Format]**
> - 请直接提供代码。
> - 不要包含 ```java 外的任何冗长废话或寒暄。
> - 在函数上补充 Javadoc 注释。

你看，后者不仅给了语言限定，交代了业务场景里的隐藏约束（处理判空、时区转换），还非常职业地去掉了 AI 爱讲的“好嘞，这就为您生成”这些废话。

## 2. Few-Shot Prompting (少样本提示)

我们常说的 RCTF 属于 **0-shot（零样本）** 提示。如果我们要求的格式非常奇特，或者业务逻辑极度专门化，光靠文字描述 AI 很难抓准。

这个时候，**“给它抄作业”** 是最有效的策略。我们在提示词里面直接给它 1 到 2 个完成好的输入输出示例（Few-Shot）。

**举个例子：**

> 我需要你从非结构化的客服对话录音文字本中，提取出用户的核心诉求类别，并打上标签。
> 
> 示例 1：
> 文本："喂，我昨天买的这双鞋鞋带断了你们不管管吗？"
> 标签：【质量投诉】
> 
> 示例 2：
> 文本："麻烦查一下我的那个尾号 3324 的单子发货没啊？"
> 标签：【物流进度查询】
> 
> 现在，请对以下文本进行打标：
> 文本："我不想用了，把上次借的钱怎么给你们补回去？"
> 标签：

通过给它喂“示例”，模型会在自己的注意力机制中瞬间捕获你要的模式映射规律。**Many times, showing is better than telling. (展示往往胜于说教)**

## 3. Think Step-by-Step (思维链技术 - CoT)

这是让大模型解决复杂逻辑、数学推理或者繁杂代码重构的终极杀器：**Chain of Thought (CoT)**。

当你丢给 AI 极其复杂的算法时，不要让它直接给你最终结果或者最终代码。由于只能单向顺次预测 Token，它极有可能在刚开头就预测偏了逻辑，导致后面的代码全面崩盘。

**正确做法：强制要求它分步骤思考和输出。**

> 请帮我解决上面那个复杂的订单分库分表数据迁移逻辑。**在我们开始写代码前，请你先一步一步用中文列出你的迁移思路和注意事项。**

通过让它“大声说出想法”，它生成的每一步中间推论过程，实际上都变为了它下一阶段输出代码的绝佳 **Context (上下文)**！这就是变相放大了模型的推理能力。

---

## 4. 迭代与调优：没有一次成型的 Prompt

不要指望写出一个惊为天人的 Prompt 然后就永远一劳永逸。
当发现大模型的输出效果不佳时，按照以下路径反思你的提示词：
1. **指令是否模糊交织？** 如果存在两件复杂的事，请拆开，先聊完第一件事，利用下一轮对话聊第二件事。
2. **是否隐含了假设？** 你脑子里知道这是前端 React 项目，但你忘了在提示词里说，导致它给你输出了原生 JS。
3. **增加“反向约束”**：在 Format 环节告诉它 **“绝对不要”** 怎样做（如“绝对不要使用 `any` 类型”）。
