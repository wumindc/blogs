# 解密：“模型 (Model)”与“参数 (Parameters)”到底是什么？

> 当我们在谈论 GPT-4、Llama 3 或者是通义千问时，我们总说“它是一个拥有千亿参数的大模型”。作为一个写业务代码的程序员，第一次听到“模型”这个词，脑子里往往是一团浆糊——它到底是一个 Java Class？还是一个巨大的 JSON 配置文件？还是嵌在某个 `while` 循环里的一大串 `if-else`？

揭开覆盖在“人工智能”上的神秘面纱，本节我们将用最直白的语言来打破这个黑盒。

---

## 1. 放弃基于“逻辑规则”的错觉
首先，必须在大脑里把传统写代码的 `if(a == b) { return c; }` 这种思维彻底删掉。
深度学习构建出的 AI 模型，内部**没有任何一条**显式的业务规则代码。它不包含任何一句人类写下的类似于“如果别人问中国首都是哪里，你就回答北京”的判定。

## 2. “模型”的物理真身：一堆浮点数矩阵

既然没有代码逻辑，那大模型跑起来的核心到底是什么？
答案非常枯燥且不可思议：大模型的核心，在硬盘上表现为 **一个极其巨大的二进制文件（可能是几百个 GB 大小），里面装满了密密麻麻的数字（确切地说，是浮点小数，比如 `0.452`, `-1.238`, `0.009`）。**

这就是**模型 (Model)**。

这些极其繁杂的小数被组织成了极其复杂的立体矩阵，我们称这些数字为 **参数 (Parameters) / 权重 (Weights)**。

*   当你说下载了一个 7B（70亿参数）的开源大模型时，你其实就是下载了一个装着 70 亿个浮点数的大文件。
*   当你说这个模型占用了你 14GB 的显存时，是因为每个浮点数如果用 16位精度 (FP16/BF16，即 2 个字节) 存储，70亿个数字刚好就是大约 14 个 GB。

## 3. 它是如何“思考”的？(简单的数学直觉)

想象一个极其简单的神经元：`y = w1 * x1 + w2 * x2 + b`
*   `x` 就是输入进去的数据（比如代表用户问题的一个个字的特征值）。
*   `w` 和 `b` 就是我们刚才提那几百亿个**参数/权重**。
*   `y` 就是算出来的结果概率。

当你问 GPT 一个问题时，你的这串文字被转换成了一串数字输入 `x` 矩阵，然后和模型中那高达上千亿规模代表权重的 `w` 矩阵进行巨量的**矩阵乘法心算**。在这近百层的神经网络乘加运算一层层像瀑布一样冲刷下来后，最底层流出来的 `y` 矩阵，就代表了下一个生成的字，大概率应该是哪一个字。

所以，大模型内部的推理过程，本质就是**在做海量的乘法和加法运算**。这也是为什么 CPU 算不动它，必须要用擅长并行乘加运算的 GPU 来跑它的原因。

---

## 4. 参数是怎么来的？—— 炼丹术（Training）

你一定会问：那这装着 70 亿个数的巨大矩阵里，每个数字到底填多少？是 0.3 还是 -0.5？谁来填它？

这就引出了 **训练 (Training)** 这个词。由于模型原理太像一个魔法黑盒，所以业界戏称为 **“炼丹”**。

1.  **初始状态**：如果刚建好一个模型架构，那这 70 亿个数字全是一盘散沙（随机生成的数字）。你问它“你好吗”，它可能会预测输出一句毫无意义的“猪猪红”。
2.  **暴力喂数据**：科学家把全人类的维基百科、Reddit 论坛帖子、GitHub 几十年的代码库作为“教材”。用 GPU 并行算出结果，如果算错下一个字，就用一种名为**反向传播 (Backpropagation)** 的微积分算法，去微微地调教这 70 亿个数字的大小。
3.  **最终成型（涌现能力）**：经过大半年时间，耗费几千万美金的电费和几万张 A100 显卡。这 70 亿个数字终于被“平衡”到了一个极其绝妙的概率状态。在这个极其玄学的分布下，它实际上“隐式地压缩”了全人类的常识和语言规律。

这个时候，这个装满“调优之后”数字的大文件，就出山了，我们称它为**预训练大模型 (Pre-trained Foundation Model)**。

---

## 小结

所以下一次，当你向 API 发送一个请求，得到一段优美的 C++ 代码或者一篇优美的散文时，请怀着崇敬的心情：

**这并不是它从某个库里查出来的代码片段。这是它通过这千亿个被称为“参数”的浮点数，进行了一场极其宏大、精准且神秘的矩阵运算，即时为你接龙算出来的心血之作。**

不过在这个过程中，它其实根本不认识中文和英文字母。那在我们人类看来的文字段落，在进入矩阵相乘之前，又被处理成谁了呢？下一章揭晓：**[什么是 Token 与上下文窗口？](./what-is-token)**。
