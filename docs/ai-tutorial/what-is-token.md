# 深入数字世界的微粒：什么是 Token 与上下文窗口？

> 作为后端或前端工程师，如果你打算在代码里接入大语言模型的 API，你最需要搞懂也是最直接与“钱”挂钩的一个概念，就是 **Token (词元)**。
> 大模型不认识汉字，也不懂英文字母，它是一个只吃数字字面的纯理科巨兽。Token，就是它的唯一精神食粮。

## 1. 词元化 (Tokenization)：人类语言的“切片机制”

假设你是一个模型，你的几千亿个神经网络在等待计算，而用户在对话框里输入了一句：
`"I love eating hamburgers, but ChatGPT doesn't eat anything."`

在把这句话丢进模型进行矩阵相乘之前，必须有一个前置的翻译官将它“切碎”并映射成查表数字。这个过程就叫做 Tokenization。
在经典的切词算法（如 BPE 算法）下，长单词会被拆开，短单词会保留，标点符号也会独立出来：

*   `"I"` -> 1 个 token
*   `" love"` -> 1 个 token (注意：经常包含前面的空格)
*   `" eating"` -> 1 个 token
*   `" hamburgers"` -> 可能会被切碎成 3 个 token：`" ham"`, `"bur"`, `"gers"`
*   `","` -> 1 个 token
*   `" but"` -> 1 个 token
*   `" ChatGPT"` -> 可能会被切碎成 `" Chat"`, `"G"`, `"PT"`
*   ...

### 为什么不按个拆成单独的字母（比如 h, a, m, b, u, r, g, e, r）？
因为如果拆得太细，模型看到的就只是一堆没有关联的单字母，它很难马上联想到“这是一种食物叫汉堡包”，且这会导致这段话变得极度冗长，浪费计算资源。

### 为什么不把一整句话或整个单词作为一个单位？
因为世界上可能出现无限种生僻词或网络新词的组合，你无法在一个固定大小的“词典”里穷举所有的可能。而像 `"Chat"`, `"G"`, `"PT"` 这样的基本字根/词缀组合是非常通用且有限的。这就保证了模型就算遇到极其陌生的词，也能拼装认出来。

### 经验法则（对于算费极其重要）
1.  **英文环境**：1 个 Token 大约等于 0.75 个英文单词。
2.  **中文环境**：中文切词比较吃亏，1 个汉字通常会被切成 1 到 2 个 Token 甚至有时候是 3 个（因为底层是基于 UTF-8 编码字节切分）。这就是为什么同样问一段话，你用中文 API 往往比英文要消耗更多配额。
3.  **代码环境**：代码也是按照相应的变量名、符号切割。空格和缩进在很多词表下也会占用 Token。

## 2. 上下文窗口 (Context Window)：大模型的“短期记忆容量”

前面我们讲了，大模型是没有记忆的。如果你和它聊了 50 轮，为了让它“记住”你们前面讨论的内容，每次请求 API 时，你必须把你俩前面的所有历史对话（Message 数组），包含最新的提问，一股脑全部传给大模型。

这就是为什么我们会引出 **上下文窗口** 这个概念。

*   **它相当于人类的“工作台面”或“短期记忆（RAM）大小”**。
*   早期的 GPT-3 上下文窗口只有 `4K` (约 4,000 个 Token，大概三千个汉字)，如果你长篇大论超过了这个数字，模型就会出现报错或者截断，前面说的话它就不记得了。
*   随后技术突破，很快飙升到 `16K`、`32K`。
*   如今的当打之年，GPT-4-Turbo 或 Claude 3 拥有 `128K` 甚至 `200K` 的上下文。而国产的佼佼者（如 Kimi 或是通义千问升级版）宣称能达到 `100万甚至200万` 的超长文本处理能力。

### 大不一定就是好：长文本的两大痛点
虽然有的模型宣称可以扔几十万字（相当于一本《三体》）进去，但：
1. **“海里捞针 (Needle in a Haystack)”退化**：由于自注意力机制在算千层矩阵，丢进去的书太长，模型往往只能记住开头和结尾的内容，丢在中段的关键知识点极容易被忽视或遗忘（幻觉发作）。
2. **极端的烧钱与高延迟**：对于大部分厂商而言，你每次提问时塞入 10 万个 Token 获取答案，和塞入 1 千个 Token，对服务器造成的算力开销是指数级增长的。而且 API 通常按照发送的 Token 计费，如果你每一轮聊天都要发一本《三体》过去，你的钱包很快就会被烧穿。

这就是为什么我们会在 AI 原生应用开发后期，引入 **RAG (检索增强) 架构**。我们宁愿先自己在数据库里搜出关于这个问题最相关的两页 A4 纸文字，再把这两页文字当作仅有的 `2K Tokens` 塞给大模型的短期记忆（上下文），让它精准总结。我们不鼓励也不舍得生猛地“把整个公司的文档全喂给它”。
